## ✅ Verification Checklist - ประวัติการแก้ไข

ตรวจสอบให้แน่ใจว่าระบบติดตั้งเสร็จและทำงานอย่างถูกต้อง

### 1️⃣ ตรวจสอบไฟล์ที่เพิ่มหรือแก้ไข

- [ ] **API Endpoint สร้างแล้ว**
  ```
  src/app/api/orders/[id]/history/route.ts
  ```
  - ตรวจสอบ: ไฟล์มีหรือไม่?

- [ ] **EditHistory Component สร้างแล้ว**
  ```
  src/app/printer/components/EditHistory.tsx
  ```
  - ตรวจสอบ: ไฟล์มีหรือไม่?

- [ ] **Dashboard อัปเดตแล้ว**
  ```
  src/app/printer/dashboard/page.tsx
  ```
  - ตรวจสอบ: มี import EditHistory หรือไม่?
  - ตรวจสอบ: มี logAuditTrail() function หรือไม่?
  - ตรวจสอบ: ที่แสดง edit history มีการใช้ <EditHistory> component หรือไม่?

### 2️⃣ ทดสอบในหน้า Dashboard

**ขั้นที่ 1: เข้าสู่ Dashboard**
- [ ] เข้าได้ปกติไม่มี error ❌

**ขั้นที่ 2: หาคำสั่งที่มีการแก้ไข**
- [ ] เลือกคำสั่งที่มีการแก้ไขมาก่อน
- [ ] ดูว่าในการ์ด มี "ประวัติการแก้ไข" ปุ่มหรือไม่

**ขั้นที่ 3: ขยายประวัติ**
- [ ] คลิกปุ่ม "ประวัติการแก้ไข" 
- [ ] ควรเห็นข้อมูล:
  - [ ] ชื่อ Action (สร้าง, แก้ไข, ตรวจสอบ, ยกเลิก)
  - [ ] เวลา (วันเวลาไทย)
  - [ ] ชื่อผู้ทำการแก้ไข
  - [ ] สรุปการเปลี่ยนแปลง

**ขั้นที่ 4: ทดสอบการแก้ไขใหม่**
- [ ] แก้ไขคำสั่งใดคำสั่งหนึ่ง
- [ ] บันทึกการแก้ไข
- [ ] ขยายประวัติ
- [ ] ควรเห็นการแก้ไขล่าสุดด้านบนสุด
- [ ] ควรเห็น "ล่าสุด" badge ที่การแก้ไขที่เพิ่งเสร็จ

### 3️⃣ ทดสอบการบันทึก Audit Log

**ทดสอบหลังแก้ไข:**
- [ ] แก้ไข: Quantity
- [ ] แก้ไข: Lot Number  
- [ ] แก้ไข: Production Date
- [ ] แก้ไข: Notes

**ทดสอบหลังตรวจสอบ:**
- [ ] กด "ตรวจสอบ" ปุ่ม
- [ ] ดูประวัติควรเห็น "ตรวจสอบ" action

**ทดสอบหลังยกเลิก:**
- [ ] กด "ยกเลิก" ปุ่ม
- [ ] ระบุเหตุผล
- [ ] ดูประวัติควรเห็น "ยกเลิก" action

### 4️⃣ ทดสอบ Browser Console

ไม่ควรมี Error แดงใดๆ:

1. **เปิด DevTools:** `F12` หรือ `Cmd+Option+J`
2. **ไปที่ Console tab**
3. **ลองขยาย/ยุบประวัติ**
4. **ตรวจสอบ:**
   - [ ] ไม่มี Error messages ❌
   - [ ] API request สำเร็จ (ดูใน Network tab)

### 5️⃣ SQL Setup (ถ้าต้องการ)

ถ้าต้องการให้ tracking ที่ดียิ่งขึ้น:

- [ ] เปิด Supabase Console
- [ ] ไปที่ SQL Editor
- [ ] คัดลอก `setup_enhanced_audit_logs.sql`
- [ ] รันให้สำเร็จ ✅
- [ ] ตรวจสอบ Table `audit_logs` ถูกสร้าง
- [ ] ทดสอบแก้ไขใหม่ - บันทึกควรไปใน audit_logs

### 6️⃣ Performance Check

- [ ] Dashboard โหลดเร็ว (< 3 วินาที)
- [ ] ประวัติขยายไม่ชา (< 1 วินาที)
- [ ] ไม่มี lag หรือตัวอักษรหลุด
- [ ] Icons แสดงถูกต้อง ✅

### 7️⃣ Edge Cases (กรณีพิเศษ)

- [ ] คำสั่งที่ไม่เคยแก้ไข - ไม่มี "ประวัติการแก้ไข" ปุ่ม ✓
- [ ] ประวัติที่เยอะ - สามารถเลื่อนดูได้ ✓
- [ ] ไม่มี audit_logs table - ยังคงใช้ได้ (fallback) ✓
- [ ] API error - แสดง error message ที่เข้าใจ ✓
